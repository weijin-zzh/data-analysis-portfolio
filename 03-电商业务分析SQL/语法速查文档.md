# 电商业务分析SQL项目 - 语法速查文档

## 一、基础查询语法

### 1. SELECT语句（数据查询）

```sql
-- 基本查询
SELECT 列名1, 列名2 FROM 表名;

-- 查询所有列
SELECT * FROM 表名;

-- 去重查询
SELECT DISTINCT 列名 FROM 表名;

-- 取别名
SELECT 列名 AS 别名 FROM 表名;
```

### 2. WHERE子句（条件筛选）

```sql
-- 基本条件
SELECT * FROM 表名 WHERE 列名 = '值';

-- 范围查询
SELECT * FROM 表名 WHERE 列名 BETWEEN 值1 AND 值2;

-- 不等于
SELECT * FROM 表名 WHERE 列名 != '值';
SELECT * FROM 表名 WHERE 列名 <> '值';

-- NULL值判断
SELECT * FROM 表名 WHERE 列名 IS NULL;
SELECT * FROM 表名 WHERE 列名 IS NOT NULL;
```

### 3. ORDER BY子句（排序）

```sql
-- 升序排序（默认）
SELECT * FROM 表名 ORDER BY 列名;

-- 降序排序
SELECT * FROM 表名 ORDER BY 列名 DESC;

-- 多列排序
SELECT * FROM 表名 ORDER BY 列名1 DESC, 列名2 ASC;
```

### 4. LIMIT子句（限制结果）

```sql
-- 取前N条记录
SELECT * FROM 表名 LIMIT 10;

-- 分页查询（跳过M条，取N条）
SELECT * FROM 表名 LIMIT 10 OFFSET 20;
```

## 二、聚合函数与分组

### 1. 常用聚合函数

```sql
-- 计数（去重）
COUNT(DISTINCT user_id)

-- 求和
SUM(total_amount)

-- 平均值
AVG(price)

-- 最大值
MAX(amount)

-- 最小值
MIN(amount)

-- 四舍五入
ROUND(value, 2)  -- 保留2位小数
```

### 2. GROUP BY子句（分组）

```sql
-- 按单列分组
SELECT 分组列, COUNT(*) FROM 表名 GROUP BY 分组列;

-- 按多列分组
SELECT 列1, 列2, SUM(金额) FROM 表名 GROUP BY 列1, 列2;
```

### 3. HAVING子句（分组后筛选）

```sql
-- 筛选分组结果
SELECT 分组列, COUNT(*) 
FROM 表名 
GROUP BY 分组列 
HAVING COUNT(*) > 100;
```

## 四、窗口函数（高级SQL）

### 1. 基础语法

```sql
-- 窗口函数基本格式
函数名() OVER (PARTITION BY 分组列 ORDER BY 排序列)

-- 示例：计算累计和
SUM(amount) OVER (PARTITION BY user_id ORDER BY order_time)
```

### 2. 常用窗口函数

#### 2.1 排名函数

```sql
-- 普通排名（相同值并列，后续跳跃）
RANK() OVER (ORDER BY 列名 DESC)

-- 密集排名（相同值并列，后续不跳跃）
DENSE_RANK() OVER (ORDER BY 列名 DESC)

-- 行号（连续编号）
ROW_NUMBER() OVER (ORDER BY 列名 DESC)
```

#### 2.2 聚合窗口函数

```sql
-- 移动平均
AVG(amount) OVER (ORDER BY date ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)

-- 累计求和
SUM(amount) OVER (PARTITION BY category ORDER BY date)

-- 分组内占比
amount / SUM(amount) OVER (PARTITION BY category) * 100
```

#### 2.3 偏移函数

```sql
-- 上一行数据
LAG(列名, 1) OVER (ORDER BY 排序列)

-- 下一行数据
LEAD(列名, 1) OVER (ORDER BY 排序列)

-- 第一行
FIRST_VALUE(列名) OVER (PARTITION BY 分组列 ORDER BY 排序列)

-- 最后一行
LAST_VALUE(列名) OVER (PARTITION BY 分组列 ORDER BY 排序列)
```

## 五、CTE表达式（WITH子句）

### 1. 基础语法

```sql
-- 单CTE
WITH cte_name AS (
    SELECT 列1, 列2 FROM 表名 WHERE 条件
)
SELECT * FROM cte_name;
```

### 2. 多CTE

```sql
-- 多个CTE
WITH 
    cte1 AS (
        SELECT 列1, 列2 FROM 表1 WHERE 条件1
    ),
    cte2 AS (
        SELECT 列3, 列4 FROM 表2 WHERE 条件2
    )
SELECT * FROM cte1 JOIN cte2 ON cte1.列1 = cte2.列3;
```

### 3. 项目示例：次日留存率

```sql
WITH daily_new_users AS (
    -- 每日新增用户数
    SELECT
        DATE(register_time) AS register_date,
        COUNT(DISTINCT user_id) AS new_user_count
    FROM user_master
    WHERE register_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE()
    GROUP BY DATE(register_time)
),
retained_users AS (
    -- 次日留存用户数
    SELECT
        DATE(u.register_time) AS register_date,
        COUNT(DISTINCT u.user_id) AS retained_count
    FROM user_master u
    WHERE
        u.register_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE()
        AND u.last_login_time >= DATE_ADD(DATE(u.register_time), INTERVAL 1 DAY)
    GROUP BY DATE(u.register_time)
)
-- 计算留存率
SELECT
    d.register_date,
    d.new_user_count,
    r.retained_count,
    ROUND(r.retained_count / d.new_user_count * 100, 2) AS retention_rate
FROM daily_new_users d
LEFT JOIN retained_users r ON d.register_date = r.register_date;
```

## 六、多表连接（JOIN）

### 1. 内连接（INNER JOIN）

```sql
-- 只返回两表匹配的行
SELECT * 
FROM 表1 
INNER JOIN 表2 ON 表1.关联列 = 表2.关联列;
```

### 2. 左连接（LEFT JOIN）

```sql
-- 返回左表所有行，右表匹配的行
SELECT * 
FROM 表1 
LEFT JOIN 表2 ON 表1.关联列 = 表2.关联列;
```

### 3. 右连接（RIGHT JOIN）

```sql
-- 返回右表所有行，左表匹配的行
SELECT * 
FROM 表1 
RIGHT JOIN 表2 ON 表1.关联列 = 表2.关联列;
```

### 4. 全连接（FULL JOIN）

```sql
-- 返回两表所有行
SELECT * 
FROM 表1 
FULL JOIN 表2 ON 表1.关联列 = 表2.关联列;
```

### 5. 自连接（SELF JOIN）

```sql
-- 表与自身连接
SELECT a.user_id, a.order_time, b.order_time AS next_order_time
FROM orders a
LEFT JOIN orders b ON a.user_id = b.user_id AND b.order_time > a.order_time;
```

## 八、业务指标计算示例

### 1. 用户运营指标

#### 1.1 新增用户数

```sql
-- 统计最近30天新增用户（去重）
SELECT
    COUNT(DISTINCT user_id) AS 新增用户数
FROM user_master
WHERE register_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE();
```

#### 1.2 月活跃用户数（MAU）

```sql
-- 统计当月活跃用户（有登录行为）
SELECT
    COUNT(DISTINCT user_id) AS 月活跃用户数
FROM user_master
WHERE last_login_time BETWEEN DATE_FORMAT(CURDATE(), '%Y-%m-01') AND CURDATE();
```

#### 1.3 用户次日留存率

```sql
WITH daily_new_users AS (
    SELECT DATE(register_time) AS register_date, COUNT(DISTINCT user_id) AS new_user_count
    FROM user_master
    WHERE register_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE()
    GROUP BY DATE(register_time)
),
retained_users AS (
    SELECT DATE(u.register_time) AS register_date, COUNT(DISTINCT u.user_id) AS retained_count
    FROM user_master u
    WHERE u.register_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE()
        AND u.last_login_time >= DATE_ADD(DATE(u.register_time), INTERVAL 1 DAY)
    GROUP BY DATE(u.register_time)
)
SELECT d.register_date, d.new_user_count, r.retained_count,
       ROUND(r.retained_count / d.new_user_count * 100, 2) AS retention_rate
FROM daily_new_users d
LEFT JOIN retained_users r ON d.register_date = r.register_date;
```

#### 1.4 用户渠道转化率

```sql
-- 各渠道注册用户数及转化率
SELECT 
    channel,
    COUNT(DISTINCT user_id) AS 注册用户数,
    SUM(CASE WHEN order_count > 0 THEN 1 ELSE 0 END) AS 购买用户数,
    ROUND(SUM(CASE WHEN order_count > 0 THEN 1 ELSE 0 END) / COUNT(DISTINCT user_id) * 100, 2) AS 转化率
FROM user_master u
LEFT JOIN (SELECT user_id, COUNT(*) AS order_count FROM orders GROUP BY user_id) o 
    ON u.user_id = o.user_id
GROUP BY channel;
```

#### 1.5 客单价

```sql
-- 平均每单金额
SELECT 
    ROUND(AVG(total_amount), 2) AS 客单价
FROM orders
WHERE status = 'paid';
```

### 2. 商品管理指标

#### 2.1 商品分类占比

```sql
-- 各分类商品数量及占比
SELECT 
    category,
    COUNT(*) AS 商品数量,
    ROUND(COUNT(*) / (SELECT COUNT(*) FROM products) * 100, 2) AS 占比
FROM products
GROUP BY category
ORDER BY COUNT(*) DESC;
```

#### 2.2 商品毛利润

```sql
-- 计算商品毛利润
SELECT 
    product_id,
    product_name,
    (sale_price - cost_price) * sales_count AS 毛利润
FROM products;
```

### 3. 销售分析指标

#### 3.1 GMV（商品交易总额）

```sql
-- 每日GMV统计
SELECT
    DATE(order_time) AS order_date,
    SUM(total_amount) AS daily_gmv
FROM order_master
WHERE order_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE()
GROUP BY DATE(order_time)
ORDER BY order_date;
```

#### 3.2 实际销售额（有效GMV）

```sql
-- 已支付订单金额
SELECT
    DATE(order_time) AS order_date,
    SUM(total_amount) AS daily_sales
FROM order_master
WHERE order_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE()
    AND status = 'paid'
GROUP BY DATE(order_time)
ORDER BY order_date;
```

#### 3.3 支付转化率

```sql
-- 支付订单数 / 总订单数
SELECT 
    COUNT(*) AS 总订单数,
    SUM(CASE WHEN status = 'paid' THEN 1 ELSE 0 END) AS 支付订单数,
    ROUND(SUM(CASE WHEN status = 'paid' THEN 1 ELSE 0 END) / COUNT(*) * 100, 2) AS 支付转化率
FROM orders;
```

#### 3.4 复购率

```sql
-- 购买2次及以上的用户占比
WITH user_order_count AS (
    SELECT user_id, COUNT(*) AS order_count
    FROM orders
    WHERE order_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE()
    GROUP BY user_id
)
SELECT 
    COUNT(DISTINCT user_id) AS 总购买用户数,
    SUM(CASE WHEN order_count >= 2 THEN 1 ELSE 0 END) AS 复购用户数,
    ROUND(SUM(CASE WHEN order_count >= 2 THEN 1 ELSE 0 END) / COUNT(DISTINCT user_id) * 100, 2) AS 复购率
FROM user_order_count;
```

#### 3.5 订单量

```sql
-- 每日订单数
SELECT
    DATE(order_time) AS order_date,
    COUNT(*) AS daily_orders
FROM order_master
WHERE order_time BETWEEN DATE_SUB(CURDATE(), INTERVAL 30 DAY) AND CURDATE()
GROUP BY DATE(order_time)
ORDER BY order_date;
```

### 4. 订单履约指标

#### 4.1 退货率

```sql
-- 退货订单数 / 总订单数
SELECT 
    COUNT(*) AS 总订单数,
    SUM(CASE WHEN is_refunded = 1 THEN 1 ELSE 0 END) AS 退货订单数,
    ROUND(SUM(CASE WHEN is_refunded = 1 THEN 1 ELSE 0 END) / COUNT(*) * 100, 2) AS 退货率
FROM orders;
```

#### 4.2 平均订单时效

```sql
-- 从发货到签收的平均时长（天）
SELECT 
    ROUND(AVG(DATEDIFF(sign_time, ship_time)), 2) AS 平均订单时效
FROM orders
WHERE ship_time IS NOT NULL AND sign_time IS NOT NULL;
```

#### 4.3 订单取消率

```sql
-- 取消订单数 / 总订单数
SELECT 
    COUNT(*) AS 总订单数,
    SUM(CASE WHEN status = 'cancelled' THEN 1 ELSE 0 END) AS 取消订单数,
    ROUND(SUM(CASE WHEN status = 'cancelled' THEN 1 ELSE 0 END) / COUNT(*) * 100, 2) AS 取消率
FROM orders;
```

### 5. 支付管理指标

#### 5.1 支付方式占比

```sql
-- 各支付方式订单数及金额占比
SELECT 
    payment_method,
    COUNT(*) AS 订单数,
    SUM(total_amount) AS 支付金额,
    ROUND(SUM(total_amount) / (SELECT SUM(total_amount) FROM orders WHERE status = 'paid') * 100, 2) AS 金额占比
FROM orders
WHERE status = 'paid'
GROUP BY payment_method;
```

#### 5.2 支付成功率

```sql
-- 成功支付订单数 / 总支付尝试数
SELECT 
    COUNT(*) AS 总支付尝试数,
    SUM(CASE WHEN payment_status = 'success' THEN 1 ELSE 0 END) AS 成功支付数,
    ROUND(SUM(CASE WHEN payment_status = 'success' THEN 1 ELSE 0 END) / COUNT(*) * 100, 2) AS 支付成功率
FROM payment_logs;
```

## 九、SQL语法对照表

### 1. Pandas vs SQL

| 操作 | Pandas语法 | SQL语法 |
|------|-----------|---------|
| 筛选行 | `df[df['列'] > 值]` | `WHERE 列 > 值` |
| 分组聚合 | `df.groupby('列').agg({'列': 'sum'})` | `GROUP BY 列 SUM(列)` |
| 排序 | `df.sort_values(by='列')` | `ORDER BY 列` |
| 去重 | `df.drop_duplicates()` | `DISTINCT` |
| 连接 | `pd.merge(df1, df2)` | `JOIN` |
| 统计 | `df['列'].mean()` | `AVG(列)` |
| 前N行 | `df.head(N)` | `LIMIT N` |

### 2. 窗口函数 vs 聚合函数

| 特性 | 聚合函数 | 窗口函数 |
|------|---------|---------|
| 结果行数 | 每组一行 | 每行都有结果 |
| GROUP BY | 需要 | 不需要 |
| 保留明细 | 不保留 | 保留 |
| 适用场景 | 汇总统计 | 明细+排名/累计 |

## 十、SQL性能优化建议

### 1. 使用索引

```sql
-- 创建索引
CREATE INDEX idx_user_id ON orders(user_id);
CREATE INDEX idx_order_time ON orders(order_time);

-- 复合索引（用于联合查询）
CREATE INDEX idx_user_time ON orders(user_id, order_time);
```

### 2. 避免全表扫描

```sql
-- ❌ 避免：SELECT COUNT(*) FROM orders;
-- ✅ 推荐：使用索引列
SELECT COUNT(user_id) FROM orders;

-- ❌ 避免：在WHERE子句中对列进行函数操作
WHERE DATE(order_time) = '2025-06-01';
-- ✅ 推荐：直接比较
WHERE order_time BETWEEN '2025-06-01 00:00:00' AND '2025-06-01 23:59:59';
```

### 3. 合理使用JOIN

```sql
-- 小表驱动大表
SELECT * FROM small_table s
JOIN large_table l ON s.id = l.small_id;
```

### 4. LIMIT优化

```sql
-- 分页查询时使用索引
SELECT * FROM orders ORDER BY order_time LIMIT 100 OFFSET 100000;
```

## 十一、常用业务指标公式

| 指标 | 公式 |
|------|------|
| 新增用户数 | COUNT(DISTINCT user_id) WHERE register_time IN 统计周期 |
| 活跃用户数 | COUNT(DISTINCT user_id) WHERE last_login_time IN 统计周期 |
| 次日留存率 | (次日登录的新增用户数 / 当日新增用户数) × 100% |
| 7日留存率 | (第7日登录的新增用户数 / 当日新增用户数) × 100% |
| GMV | SUM(total_amount)（含未支付订单） |
| 实际销售额 | SUM(total_amount) WHERE status = 'paid' |
| 支付转化率 | (支付订单数 / 总订单数) × 100% |
| 复购率 | (购买≥2次的用户数 / 总购买用户数) × 100% |
| 客单价 | AVG(total_amount) WHERE status = 'paid' |
| 退货率 | (退货订单数 / 总订单数) × 100% |

## 十二、数据库表结构设计

### 1. 用户主表（user_master）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| user_id | BIGINT | 用户唯一标识 |
| register_time | DATETIME | 注册时间 |
| last_login_time | DATETIME | 最后登录时间 |
| channel | VARCHAR(50) | 注册渠道 |
| status | TINYINT | 用户状态（0禁用，1启用） |

### 2. 订单主表（order_master）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| order_id | BIGINT | 订单唯一标识 |
| user_id | BIGINT | 用户ID（外键） |
| total_amount | DECIMAL(10,2) | 订单总金额 |
| order_time | DATETIME | 下单时间 |
| status | VARCHAR(20) | 订单状态（pending/paid/shipped/signed/cancelled/refunded） |

### 3. 商品表（products）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| product_id | BIGINT | 商品唯一标识 |
| product_name | VARCHAR(200) | 商品名称 |
| category | VARCHAR(50) | 商品分类 |
| price | DECIMAL(10,2) | 商品价格 |
| stock | INT | 库存数量 |

### 4. 支付记录表（payment_logs）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| payment_id | BIGINT | 支付记录唯一标识 |
| order_id | BIGINT | 订单ID（外键） |
| payment_method | VARCHAR(50) | 支付方式（alipay/wechat/card） |
| amount | DECIMAL(10,2) | 支付金额 |
| payment_time | DATETIME | 支付时间 |
| payment_status | VARCHAR(20) | 支付状态（success/failed/pending） |
